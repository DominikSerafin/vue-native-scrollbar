Vuebar v2.0

The first Vue 2 custom scrollbar library that only enhances vanilla scrolling instead of reimplementing/hijacking it with custom scroll behavior. It uses native scroll events to detect and synchronize scrollbar position. This makes possible to hack into the native events of scrolling elements without hassle.

It was built by Dominik Serafin for GGather.com where it plays nicely with a lot of dynamic content changes and most importantly infinite scrolling and drag and drop features. And now after testing, fixes, improvements and added customization options - it's ready to be shared with the world.

Features

  • Directive instead of a custom wrapper component.
  • Doesn't hijack native scroll events and methods.
  • Can be used to style vertical or horizontal (new in v2.0) scrollbars.
  • Highly customizable. Simple to use.
  • Lightweight & performant. No 3rd party dependencies.
  • Works in browser and also build (webpack, etc.) environments.
  • Compatible with server-side rendered Vue applications and Nuxt.
  • Compatible with all major browsers including IE9 and above.
  • Tested in production.

Limitations

Unfortunately, after long time researching and working on it - I couldn't find a way to make Vuebar work properly (content gets partially hidden) with both vertical and horizontal scrollbars active when browser uses overlayed scrollbars (while still retaining native scroll events & methods).

Popular devices and browsers that use overlayed scrollbars:

  • Most of mobile (iOS, Android, etc.) devices and browsers.
  • Microsoft Edge, IE10 & IE11, when -ms-overflow-style: -ms-autohiding-scrollbar styling is enabled.
  • macOS, when Show scroll bars system option is set to When scrolling or to Automatically based on mouse or trackpad and mouse is used.

In such cases, as a fallback - Vuebar will disable itself on these browsers & devices.

If you know how to fix this or are willing to investigate - any contributions are welcome!

Quick Start

Installation

NPM

npm install vuebar --save

CDN

https://unpkg.com/vuebar

Manual

Download the .js file directly from latest commit or release and include manually in your project

vuebar.js

Quick Start

Alternative #1: Include in browser

<script src="vuebar.js"></script>

Alternative #2: Include in your build

import Vuebar from 'vuebar';
Vue.use(Vuebar);

  var app = new Vue({

    el: '#app',

    data: function(){
      return {
        popoverOpen: false,
        daySeconds: 0,
      }
    },

    created: function(){
      this.setDaySeconds();
    },

    mounted: function(){
      this.daySecondsInterval();
    },

    methods: {

      setDaySeconds: function(){
        var dt = new Date();
        var secs = dt.getSeconds() + (60 * dt.getMinutes()) + (60 * 60 * dt.getHours());
        this.daySeconds = secs;
      },

      daySecondsInterval: function(){
        setInterval(function () {
          this.setDaySeconds();
        }.bind(this), 1000);
      },

    },

  });
